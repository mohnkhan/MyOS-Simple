# Stage 5 — Stabilized Release (v3)

The final stage. Functionally this is `helloworld-os-c-v2` consolidated for
release: the experimental `alias`/`unalias` commands are removed (the command
set drops from 20 to 18), the command surface is locked, and the actual built
artifacts (`boot.bin`, `kernel.bin`, and the bootable `.img` images) are
committed so the thing that boots is the thing in the repository.

All subsystems carried over from v2 are byte-for-byte identical: the CMOS
real-time clock (`rtc.c`), the cooperative process model (`process.c`), the
shared kernel/VGA helpers, and the keyboard and type headers. The only source
that differs from v2 is `shell.c`, which has the alias machinery removed.

For the complete command reference, see `README_SHELL.md` in this directory.


## Position in the series

This is the head of the progression that begins with 60 lines of real-mode
assembly (`helloworld-os-asm`) and ends here with a ~1,950-line protected-mode C
system. Relative to stage 4 (`helloworld-os-c-v2`):

| | v2 (stage 4) | v3 (stage 5) |
|---|---|---|
| Commands | 20 | 18 |
| `alias` / `unalias` | present | removed |
| `shell.c` | ~1,311 lines | ~1,121 lines |
| `rtc.c`, `process.c`, headers | — | identical to v2 |
| Built artifacts committed | no | yes |


## Subsystems

Identical to stage 4. In brief:

- **Real-time clock** (`rtc.c`) — reads time/date from CMOS over ports
  `0x70`/`0x71`, converts BCD to binary, tracks uptime from a recorded boot
  timestamp. Commands: `time`, `date`, `clock`, `uptime`.
- **Process model** (`process.c`) — up to 10 processes with a PCB each
  (PID, name, state, context, stack, priority), a cooperative round-robin
  scheduler, and sample workloads (counter, Fibonacci, prime). Commands: `ps`,
  `run`, `kill`, `suspend`, `resume`.
- **Calculator** (`shell.c`) — fixed-point decimal arithmetic (scale 1000) for
  `+ - * /` without a math library. Command: `calc`.
- **Shell ergonomics** (`shell.c`) — command history and tab-completion against
  the command table. Commands: `history`, `memory`, `stats`.

See the `helloworld-os-c-v2` README for the detailed subsystem descriptions.


## Full command set (18)

```
help   clear   echo   calc    memory  stats   history  about  shutdown
ps     run     kill   suspend resume  time    date     clock  uptime
```

The two commands removed relative to v2 are `alias` and `unalias`.


## Source layout

```
helloworld-os-c-v3/
├── boot.asm           Bootloader (loads 39 sectors), GDT, switch to PM.
├── boot_simple.asm    Standalone 512-byte assembly image.
├── shell.c            Shell: input loop, dispatch, calc, history, tab-complete.
├── process.c/.h       Cooperative process model and scheduler.
├── rtc.c/.h           CMOS real-time clock and uptime.
├── kernel.c           Legacy stage-2 kernel, retained for reference (not built).
├── kernel.h           VGA helpers and shared kernel I/O inlines.
├── keyboard.h         Scancode tables and keyboard port/flag definitions.
├── stdint.h / stddef.h  Minimal freestanding type definitions.
├── kernel_entry.asm   32-bit entry stub: _start -> shell_main.
├── linker.ld          Places the kernel at 0x1000, output flat binary.
├── Makefile           Build/run/debug targets.
├── boot.bin           Committed build output: assembled boot sector.
├── kernel.bin         Committed build output: linked kernel.
├── *.o                Committed build outputs: object files.
├── helloworld-c.img       Committed bootable image (boot.bin + kernel.bin, 20480 B).
└── helloworld-simple.img  Committed bootable simple image (512 B).
```

The committed `.bin`, `.o`, and `.img` files are regenerated by `make` and can be
removed with `make clean`; they are kept in the repository so a reader can boot
the release without a toolchain.


## Boot specifics

Same as stage 4: the bootloader loads **39 sectors** to `0x1000`, sets up the
flat three-entry GDT, switches to 32-bit protected mode, sets the stack to
`0x90000`, and calls `shell_main`. The final image is padded to **20480 bytes**
(40 sectors). See the `helloworld-os-c` README for the full boot walkthrough.


## Prerequisites

```sh
sudo apt install nasm gcc gcc-multilib binutils make qemu-system-x86
```


## Build

```sh
make            # build helloworld-c.img and helloworld-simple.img
make clean      # remove *.bin *.o *.img
```

Build steps:

```sh
nasm -f bin   boot.asm         -o boot.bin
nasm -f elf32 kernel_entry.asm -o kernel_entry.o

gcc -m32 -ffreestanding -fno-pic -fno-pie -nostdlib -nostdinc \
    -fno-builtin -fno-stack-protector -nostartfiles -nodefaultlibs \
    -Wall -Wextra -c shell.c   -o shell.o
gcc ... -c process.c -o process.o
gcc ... -c rtc.c     -o rtc.o

ld -m elf_i386 -T linker.ld -o kernel.bin \
    kernel_entry.o shell.o process.o rtc.o

cat boot.bin kernel.bin > helloworld-c.img
truncate -s 20480 helloworld-c.img
nasm -f bin boot_simple.asm -o helloworld-simple.img
```

See the `helloworld-os-c` README for a flag-by-flag explanation of the compiler
options and the linker script. The same two benign `-Wextra` warnings as v2 are
expected (an unused variable in `process.c`, an unused helper in `rtc.c`).


## Run

```sh
make run          # boot the release system
make run-simple   # boot the pure-assembly simple image
make debug        # boot under QEMU with a GDB stub (-s -S)
```

First commands to try: `help`, `clock`, `uptime`, `calc 22 / 7`, `ps`.


## Notes and limitations

Identical to stage 4:

- Cooperative scheduling only; no preemption, no memory protection.
- Fixed-point calculator (three fractional digits), not IEEE-754.
- RTC reads are not synchronized against the CMOS update flag.
- `shutdown` halts the CPU; close QEMU to exit.
